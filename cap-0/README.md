# Cap 0 HTML5

## Resumen

Este capítulo presenta cinco ideas fundamentales para comprender HTML5 y comenzar a usarlo de manera práctica sin complicaciones ni la necesidad de reemplazar todo lo aprendido anteriormente.

---

## 1. HTML5 no es una sola cosa  
HTML5 no es un único conjunto monolítico de funciones, sino una colección de características independientes que puedes usar según las necesites: **canvas**, **video**, **geolocalización**, entre muchas otras.  
Más allá de las etiquetas, HTML5 también define cómo interactúan estas con JavaScript por medio del **DOM**, permitiendo detectar compatibilidad y manipular elementos como videos (reproducir, pausar, silenciar, revisar progreso, etc.).

---

## 2. No necesitas tirar nada a la basura  
HTML5 está construido sobre HTML4, por lo que no exige que vuelvas a aprenderlo todo desde cero.  
Tu código HTML4 **sigue funcionando** sin cambios.  
Sin embargo, HTML5 añade características nuevas como controles de formulario modernos: *sliders*, *date pickers* o el tipo de campo **email**, que mejora la experiencia en móviles ajustando el teclado automáticamente.

---

## 3. Es fácil comenzar  
Actualizar a HTML5 puede ser tan simple como cambiar el **doctype** a:

```html
<!DOCTYPE html>
```

Esto habilita el uso de nuevos elementos semánticos como `<article>`, `<section>`, `<header>` y `<footer>` sin afectar tu código existente.

---

## 4. Ya funciona hoy  
Los navegadores modernos como Chrome, Firefox, Safari, Opera y navegadores móviles soportan la mayoría de las funciones de HTML5:  
- Canvas  
- Video  
- Geolocalización  
- Almacenamiento local  
- Nuevos formularios  

Incluso navegadores más antiguos pueden recibir soporte parcial mediante polyfills o tecnologías previas como Flash o Gears.

---

## 5. HTML5 llegó para quedarse  
El W3C decidió en 2009 dejar de lado XHTML2 y enfocarse completamente en HTML5, confirmando que HTML5 es el estándar central de la web actual y futura.
